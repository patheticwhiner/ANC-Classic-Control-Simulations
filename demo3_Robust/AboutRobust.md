# 基于鲁棒与自适应控制的ANC实验综述

## 1. 概览

本项目探索了在主动噪声控制（Active Noise Control, ANC）背景下应用的几种关键控制策略。这些策略可以从理论上分为两大类：**鲁棒控制**与**自适应控制**。

- **鲁棒控制 (Robust Control)**：旨在设计一个**固定参数**的控制器，该控制器对模型不确定性和外部扰动具有指定的性能保证。此类方法适用于系统特性相对稳定，但存在建模误差或持续扰动的场景。
  - **`Bai1997` - H∞混合灵敏度控制**: 采用现代鲁棒控制的H∞综合方法，通过求解Riccati方程来系统地优化性能与鲁棒性之间的权衡。这是一种基于状态空间模型的频域设计方法。
  - **`Jafari_fixed2015`  带内插约束的H∞优化**：此方法解决的是一个带等式约束的H∞优化问题。目标是在保证系统对宽带扰动鲁棒（即最小化某个H∞范数）的同时，在特定频率点上精确满足性能指标（如完全消除某个频率的噪声），这表现为数学上的“内插约束”。将此问题构建为一个凸优化模型，使用CVX等工具一次性求解出最优的固定控制器参数。可视为 **鲁棒控制-凸优化方法**。
  - **`Carmona2000` - RST灵敏度函数塑形（Loop Shaping）**: 采用经典控制理论中的RST多项式控制器结构，通过直接在频域上“塑造”灵敏度函数来满足性能和鲁棒性要求。
- **自适应控制 (Adaptive Control)**：设计一个**参数可在线调整**的控制器，使其能够适应被控对象参数的变化或未知性。此类方法适用于系统特性随时间变化或初始模型不精确的场景。
  - **`Landau2005` - 基于RST的自适应极点配置**: 将RST控制器与自适应律相结合，通过在线更新多项式系数，强制闭环系统的极点位于期望位置，从而实现对系统动态特性的控制。这是一种自适应**反馈**控制策略。
  - **`Jafari_adaptive2015`  带内插约束的自适应H∞优化**：将同一问题转化为一个在线参数辨识问题，通过RLS或梯度下降等自适应算法，在时域上迭代更新控制器参数，使其逐步收敛到满足频域优化目标的解。可视为 **自适应鲁棒控制** 或 **基于参数辨识的控制器设计**。

### 理论关系总结
- **H∞混合灵敏度 (`Bai1997`)** 和 **带内插约束的H∞优化 (`Jafari2015`)** 都是H∞鲁棒控制的不同变体。前者是标准的混合灵敏度问题，后者则增加了额外的等式约束，难度更高。
- **RST灵敏度塑形 (`Carmona2000`)** 是解决鲁棒性能问题的经典多项式方法，与H∞方法并行。
- **自适应极点配置 (`Landau2005`)** 是一种纯粹的自适应反馈控制，其目标是跟踪时变动态特性，与前述的鲁棒性能优化目标不同。
- **鲁棒控制**与**自适应控制**是两种不同的设计哲学，但`Jafari`方法巧妙地结合了两者：用自适应的手段来求解一个鲁棒控制问题。

## 2. Bai1997：H∞混合灵敏度控制 (DGKF解法)

### `augw` 函数：自动构建广义对象

手动连接被控对象 `G` 和权重函数 `W1`, `W2`, `W3` 来构建广义对象 `P` 是一个繁琐且容易出错的过程。MATLAB 的 `augw` (augment weights) 函数极大地简化了这一过程。

```matlab
P = augw(G, W1, W2, W3);
```

- `G`: 你的被控对象（传递函数 `tf` 或状态空间 `ss` 模型）。
- `W1`, `W2`, `W3`: 上述定义的权重函数。如果某个权重不使用，可以用一个空的 `[]` 代替。

`augw` 会自动将这些模块连接起来，生成一个符合 `hinfsyn` 输入要求的状态空间模型 `P`。

### `hinfsyn` 函数：综合 H∞ 控制器

`hinfsyn` (H-infinity synthesis) 是执行核心优化计算的函数。它接收广义对象 `P`，并计算出最优的 H∞ 控制器 `K`。

```matlab
[K, CL, gamma] = hinfsyn(P, NMEAS, NCON);
```

- **输入**:
  - `P`: 由 `augw` 生成的广义被控对象。
  - `NMEAS`: 测量输出 `y` 的数量（即控制器 `K` 的输入数量）。对于单输入单输出（SISO）系统，`NMEAS = 1`。
  - `NCON`: 控制输入 `u` 的数量（即控制器 `K` 的输出数量）。对于 SISO 系统，`NCON = 1`。

- **输出**:
  - `K`: 计算得到的最优 H∞ 控制器。
  - `CL`: 从 `w` 到 `z` 的闭环传递函数 `Tzw`。
  - `gamma`: 最终达到的 H∞ 范数值。这是评估设计成败的关键指标。

**`gamma` 的解读**:

- `gamma < 1`: **设计成功**。这意味着所有由权重函数定义的性能/鲁棒性目标都已满足。
- `gamma >= 1`: **设计失败**。这意味着在给定的权重约束下，不存在一个稳定的控制器能够满足所有要求。此时，你需要放宽（降低）一个或多个权重函数的增益，然后重试。
